# PyDepGraph を活用した機能改修時の影響範囲分析

## 概要

複雑なPythonプロジェクトの機能改修において、PyDepGraphを活用した影響範囲調査の可能性と具体的なユースケースについて考察した結果をまとめる。

## 結論

**PyDepGraphは機能改修時の影響範囲調査において極めて高い活用可能性を持つ。**

従来のIDEや静的解析ツールでは困難だった、構造的・定量的な影響範囲分析を実現し、改修リスクの事前評価と適切な開発戦略の策定を可能にする。

## PyDepGraphの主要機能（影響範囲分析の観点）

### 1. 依存関係抽出機能
- **モジュール間依存関係**: import文による静的依存関係
- **関数レベル呼び出し関係**: AST解析による詳細な関数間依存
- **外部ライブラリ依存**: プロジェクトが使用する外部ライブラリの特定

### 2. グラフ分析機能
- **重要度分析**: PageRank的アルゴリズムによる関数・モジュールの重要度定量化
- **循環依存検出**: アーキテクチャ品質の評価
- **統計情報**: 依存関係の密度や複雑性の測定

### 3. 高速検索機能
- **Kùzuグラフデータベース**: 大規模プロジェクトでも高速な依存関係クエリ
- **多段階影響追跡**: 間接的・連鎖的な影響関係の効率的な探索

## 具体的なユースケース

### Phase 1: 改修前の計画・見積もりフェーズ

#### 1.1 変更対象の重要度評価
```bash
# 変更予定の関数の重要度を確認
uv run pydepgraph analytics importance --node-type function --filter "target_function"

# 結果例: 0.15 (高重要度) → 慎重な変更計画が必要
# 結果例: 0.01 (低重要度) → 比較的安全に変更可能
```

**活用価値**: 
- 重要度スコア 0.1以上: 高リスク、詳細な計画と広範囲テストが必要
- 重要度スコア 0.05未満: 低リスク、限定的なテストで対応可能

#### 1.2 影響範囲の階層的分析
```bash
# 1次影響: 変更対象を直接呼び出す関数
uv run pydepgraph query calls --filter "calls target_function"

# 2次影響: その関数を呼び出す関数（グラフクエリ）
# 依存関係グラフをたどって多段階の影響を効率的に追跡
```

**実用例**:
- 1次影響: 5関数 → 局所的変更
- 1次影響: 50関数 → 広範囲変更、段階的リリースを検討

#### 1.3 アーキテクチャレベルの事前チェック
```bash
# 循環依存の現状確認
uv run pydepgraph analytics cycles

# モジュール間結合度の評価
uv run pydepgraph analytics stats --node-type module
```

### Phase 2: 実装・リファクタリングフェーズ

#### 2.1 新機能追加時の構造健全性チェック
```bash
# 定期実行: 循環依存が新たに発生していないか確認
uv run pydepgraph analytics cycles

# 結合度の変化モニタリング
uv run pydepgraph analytics stats
```

#### 2.2 外部ライブラリ依存の管理
```bash
# 特定ライブラリに依存する全コードの特定
uv run pydepgraph query imports --filter "requests"

# ライブラリ置換時の影響範囲完全把握
```

**活用シナリオ**:
- ライブラリのバージョンアップ
- セキュリティ脆弱性対応
- ライセンス問題による代替ライブラリへの移行

### Phase 3: テスト・品質保証フェーズ

#### 3.1 テスト戦略の最適化
```bash
# 変更による影響範囲の全関数をリストアップ
uv run pydepgraph query functions --filter "affected_by_change"
```

**テスト優先度の決定**:
1. **高優先度**: 重要度スコアの高い関数
2. **中優先度**: 変更から2段階以内の影響範囲
3. **低優先度**: 3段階以上の間接的影響

#### 3.2 回帰テスト範囲の科学的決定
従来の経験則ベースではなく、依存関係グラフに基づく客観的な回帰テスト範囲の決定

## 従来ツールとの比較優位性

### vs. IDEの参照検索（Find Usages）

| 項目 | IDE参照検索 | PyDepGraph |
|-----|------------|------------|
| **分析深度** | 1段階（直接参照のみ） | n段階（連鎖的影響追跡） |
| **全体俯瞰** | 個別の参照点 | プロジェクト全体の構造 |
| **定量評価** | なし | 重要度スコアによる客観的評価 |
| **検索速度** | プロジェクトサイズに依存 | グラフDBによる高速検索 |

### vs. 静的解析ツール（Pylint、MyPy等）

| 項目 | 静的解析ツール | PyDepGraph |
|-----|-------------|------------|
| **分析目的** | コード品質・型安全性 | 構造・関係性分析 |
| **影響範囲分析** | 限定的 | 専門特化 |
| **アーキテクチャ分析** | 不可 | 強力（循環依存、結合度等） |
| **定量的指標** | 品質メトリクス | 構造的重要度・影響度 |

**結論**: 相補関係にあり、併用することで包括的な分析が可能

## 実用性向上のための追加機能提案

### 1. 変更差分分析（Change Impact Analysis）
```bash
# 提案機能
uv run pydepgraph evolution diff --from HEAD~1 --to HEAD
# → 今回の変更で新たに追加/削除された依存関係をレポート
```

### 2. 動的解析との連携
- DIコンテナによる動的依存注入の追跡
- プラグインアーキテクチャでの動的ロード関係の分析
- テスト実行時のトレース情報の統合

### 3. インタラクティブ可視化
- ブラウザベースの依存関係グラフ探索UI
- ノード選択による詳細情報表示
- 影響パスのハイライト機能

### 4. CI/CD統合
```yaml
# GitHub Actions例
- name: Impact Analysis Report
  run: |
    uv run pydepgraph analyze .
    uv run pydepgraph report --impact-summary >> $GITHUB_STEP_SUMMARY
```

## 現在の制約と対応

### 制約1: Evolution機能未実装
- **現状**: CLIにevolutionコマンドが実装されていない
- **影響**: 依存関係の時系列変化分析ができない
- **対応**: バックエンド実装済みのため、CLIコマンド追加で解決可能

### 制約2: 動的依存関係の限界
- **現状**: 静的解析のため動的な依存関係を捕捉できない
- **対応**: テスト実行時のトレース機能追加を提案

## Serenaとの比較・協業可能性分析

### Serenaとは

**Serena**は、MCP(Model Context Protocol)サーバーとして設計されたAIコーディングエージェント向けツールキットです。LSP(Language Server Protocol)統合により意味的コード解析を提供し、10以上の言語をサポートしています。

#### Serenaの主要機能
- **意味的コード解析**: LSPによるシンボルレベルでの正確なコード理解
- **多言語サポート**: Python, TypeScript, Go, Rust, Java, C#等の幅広い言語対応
- **AIエージェント統合**: MCPプロトコルを通じたAIツールとしての活用
- **精密なコード編集**: シンボル情報に基づく安全なコード変更

### PyDepGraph vs Serena 比較分析

| 項目 | PyDepGraph | Serena |
|-----|-----------|--------|
| **分析スコープ** | マクロ（プロジェクト全体構造） | ミクロ（シンボルレベル詳細） |
| **主要技術** | 静的解析 + グラフデータベース | LSP + MCP統合 |
| **対象言語** | Python特化 | 10+言語対応 |
| **分析レベル** | アーキテクチャレベル | 意味レベル |
| **得意分野** | 循環依存、重要度分析、影響範囲俯瞰 | 正確な参照追跡、AIによる編集 |
| **利用形態** | スタンドアローンCLI | AIエージェント連携 |

### 補完関係と協業シナリオ

#### 1. **理想的な補完関係**

PyDepGraphとSerenaは**競合ではなく、コード分析の異なるレイヤーを担う理想的なパートナー**です。

- **PyDepGraph**: "森全体"を俯瞰する構造的分析（マクロ視点）
- **Serena**: "一本の木"を詳細に観察する意味的分析（ミクロ視点）

#### 2. **機能改修時の協業ワークフロー**

##### Step 1: PyDepGraphによる一次スクリーニング
```bash
# 変更対象の構造的重要度を評価
uv run pydepgraph analytics importance --filter "target_function"

# 影響範囲の候補を高速抽出
uv run pydepgraph query calls --filter "target_function"
```

##### Step 2: Serenaによる詳細検証・修正
```python
# Serenaを通じたAIエージェントによる精密分析
# - PyDepGraphが抽出した候補の実際の参照関係を検証
# - 型情報や意味的文脈を考慮した影響評価
# - AIによる自動修正・リファクタリング実行
```

#### 3. **併用による相乗効果**

PyDepGraphとSerenaを併用することで以下の相乗効果を実現：

##### 段階的な分析アプローチ
```bash
# Step 1: PyDepGraphで構造的分析
uv run pydepgraph analytics importance --filter "target_function"
uv run pydepgraph query calls --filter "target_function"

# Step 2: Serenaで詳細な意味分析
# LSPを通じた正確な参照関係の確認
# 型情報や文脈を考慮した影響評価
```

##### 包括的な影響分析の実現
- PyDepGraphによる構造的リスクの定量評価
- Serenaによる意味的影響の精密検証
- 両者の結果を統合した総合的な改修計画の策定

### 使い分け指針

#### PyDepGraphが特に有効な場面
- **アーキテクチャ設計レビュー**: 依存構造の可視化・問題特定
- **大規模リファクタリング計画**: 全体影響の予測・計画立案
- **技術的負債評価**: 複雑性の定量化・ホットスポット特定
- **新人オンボーディング**: プロジェクト全体構造の理解支援

#### Serenaが特に有効な場面
- **AIによる自動コーディング**: 多言語での正確なコード生成・編集
- **インタラクティブなコード探索**: リアルタイムな定義・参照追跡
- **日常的な開発作業**: 小中規模変更の直接的影響確認
- **多言語環境**: 言語を跨いだ一貫したコード解析

### 併用による開発ワークフロー改善

#### 1. **二段階分析アプローチ**
```
機能改修プロセス
├─ Phase 1: PyDepGraphによる構造分析
│   ├─ 重要度評価 → 改修リスクの定量化
│   ├─ 影響範囲抽出 → 候補箇所のリストアップ
│   └─ アーキテクチャチェック → 循環依存等の事前確認

│   ├─ 正確な参照追跡 → 実際の依存関係確認

│   └─ 精密な編集支援 → 安全な変更実行

```

#### 2. **各フェーズでの価値**
- **計画フェーズ**: PyDepGraphの定量評価による客観的リスク判断

- **検証フェーズ**: 両ツールによる多角的な品質確認

#### 3. **開発効率の向上**
- **見積もり精度向上**: 構造的複雑度による工数予測の改善
- **変更安全性確保**: 意味的検証による回帰リスクの削減
- **品質継続的改善**: 両視点からの継続的な技術的負債管理

### 結論

PyDepGraphとSerenaは**それぞれ独立したツールとして価値を提供**します。PyDepGraphはマクロ視点での構造分析に特化し、Serenaはミクロ視点での意味分析に特化することで、開発者は用途に応じて適切なツールを選択・併用できます。

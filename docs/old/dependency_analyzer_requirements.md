# 依存関係分析ライブラリ要求仕様書

## 📋 プロジェクト概要

### プロジェクト名
**PyDepGraph** - Python プロジェクト依存関係分析・検索ライブラリ

### 目的
- Pythonプロジェクトの依存関係を自動抽出・グラフ化
- LLM（Claude Code等）による効率的な依存関係検索の実現
- 開発・保守作業における依存関係に基づいた意思決定支援

### 対象ユーザー
- ソフトウェア開発者
- アーキテクト
- DevOps エンジニア
- LLMベース開発ツール（Claude Code等）

## 🎯 ビジネス要求

### 主要な価値提案
1. **開発効率の向上**: 依存関係の即座把握による迅速な意思決定
2. **コード品質の向上**: 循環依存や高結合箇所の自動検出
3. **保守性の向上**: 変更影響範囲の事前分析による安全な改修
4. **学習支援**: プロジェクト構造の理解促進

### 解決すべき課題
- 大規模プロジェクトでの依存関係把握の困難
- 変更時の影響範囲の不明確さ
- リファクタリング時のリスク評価の困難
- LLMツールでの適切な情報取得の非効率性

## 🔧 機能要求

### 1. データ抽出機能

#### 1.1 モジュールレベル依存関係抽出
- Tachを使用したモジュール間依存関係の抽出
- インポート関係の分析
- 外部ライブラリ依存関係の識別

#### 1.2 関数レベル依存関係抽出
- Code2Flowを使用した関数間呼び出し関係の抽出
- クラス内メソッド関係の分析
- 非同期処理・コールバック関係の識別

#### 1.3 メタデータ収集
- ファイルサイズ・行数の計測
- 複雑度指標の算出

### 2. データ変換・正規化機能

#### 2.1 データ統合
- 複数ツールからの出力データの統合
- データ形式の正規化
- 重複データの排除

#### 2.2 グラフ構造変換
- リレーショナルデータからグラフデータへの変換
- ノード・エッジ属性の最適化
- 階層構造の保持

### 3. グラフデータベース管理機能

#### 3.1 Kùzuデータベース操作
- データベースの初期化・作成
- スキーマ定義・管理
- データのバルクインサート

#### 3.2 インデックス管理
- 検索性能最適化のためのインデックス作成
- クエリパフォーマンス監視

### 4. 検索・クエリ機能

#### 4.1 基本検索機能
- 関数名・クラス名・モジュール名による検索
- パターンマッチング検索
- 階層的検索（モジュール→クラス→関数）

#### 4.2 依存関係分析
- 直接依存関係の取得
- 推移的依存関係の取得

#### 4.3 パス分析
- 最短依存パスの検索
- 全パス列挙
- 依存経路の重要度分析

### 5. グラフアルゴリズム機能

#### 5.1 中心性分析
- 重要なモジュール・関数の特定
- ボトルネック箇所の識別
- 影響力の定量化

#### 5.2 コミュニティ検出
- 密結合モジュール群の発見
- 機能的グループの識別
- 分割候補の提案

#### 5.3 アーキテクチャ分析
- レイヤー違反の検出
- 設計原則の遵守状況確認
- 改善提案の生成
